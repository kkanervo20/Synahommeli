save(self: keysight_kt33000.keysight_kt33000.Kt33000StateStorage, state: int) → None

Saves instrument state in specified non-volatile location. Previously stored state in location is overwritten (no error is generated).

Parameters

state (int) – Parameter : {0|1|2|3|4}.

Returns

None

Remarks
•For 33600 Series instruments, the state files associated with this command are saved in files called STATE_0.STA through STATE_4.STA. These files are located in the Settings directory of internal memory. You can manage these files using Memory commands.
•The instrument has five non-volatile storage locations to store instrument states. Location 0 holds the instrument power down state. Use locations 1, 2, 3, and 4 to store other states. You can configure the instrument to recall the power-down state when power is restored (AutoRecall).
•From the remote interface only, you can use location 0 to store a fifth instrument state (you cannot store to this location from the front panel). However, location 0 is overwritten when power is cycled.
•States stored in memory are not affected by the Reset command.
•If you delete an arbitrary waveform from non-volatile memory after storing the instrument state, the waveform data is lost and the instrument will not output the waveform when the state is recalled; it will output the built-in "exponential rise" instead.
set_state_name(self: keysight_kt33000.keysight_kt33000.Kt33000StateStorage, name: str, state: int) → None

Sets the name for a storage location.

Parameters

  • name (str) – Name for a storage location. It should be an unquoted string of up to 12 characters. Default names are “AUTO_RECALL”, “STATE_1”, “STATE_2”, “STATE_3”, and “STATE_4”.

  • state (int) – Parameter : {0|1|2|3|4}.

Returns

None

Remarks

•You can name location 0, but the name is overwritten when power is cycled and a new power-down state is stored there.
•Same name can be assigned to different locations.
•State names are unaffected by Reset command.

Kt33000OutputChannelOutputFunctionArbitraryWaveformSRateCoupling

Access interface members using: driver.output_channels[<name_or_index>].output_function.arbitrary_waveform.s_rate.coupling
Where driver is a reference to the driver’s root interface returned by the constructor.
class Kt33000OutputChannelOutputFunctionArbitraryWaveformSRateCoupling

Kt33000OutputChannelOutputFunctionArbitraryWaveformSRateCoupling interface.

property enabled

Enables or disables sample rate coupling between channels, or allows one-time copying of one channel’s sample rate into the other channel

Access

ReadWrite

Type

bool

Remarks
•Parameter : 0 (OFF) or 1 (ON). Default 0 (OFF).
•The ON value starts sample rate coupling in the mode specified by CouplingMode.
•If the current offset or ratio, combined with the current sample rate settings, would cause either sample rate to exceed instrument specifications, the instrument will generate an error and the exceeded sample rate will clip at its maximum or minimum value.
•If setting mode to RATIO and setting RATIO to 1.0 still exceeds the specifications of either channel, an error message will be generated and the Coupling will not be turned ON.
•Both channels must be configured for Function set to Arbitrary in order to enable sample rate coupling.
property mode

Sets type of sample rate coupling to either a constant sample rate offset (OFFSet) or a constant ratio (RATio) between the channels’ sample rates.

Access

ReadWrite

Type

SRateCouplingMode

Remarks
•Parameter : Offset or Ratio. Default Ratio.
•Typical Return : Offset or Ratio
•The default RATio is 1.
•The default sample rate coupling is OFF.
•This setting applies to both channels.
property offset

Sets sample rate offset when a two-channel instrument is in sample rate coupled mode OFFSet.

Access

ReadWrite

Type

float

Remarks
•Parameter : Valid values depend on InterpolationEnabled setting. For NORMal and STEP, the range is between ±250 MSa/s (EDU33210 series and 33500 Series) or ±1 GSa/s (33600 Series). For OFF, the range is between ±62.5 MSa/s (EDU33210 series and 33500 Series) or ±250 MSa/s (33600 Series). In any case, default is 0.
•Typical Return : +8.000000000000000E+02
•When specifying OFFSet or RATio, the SOURce channel (SOURce1 or SOURce2) is used as the reference channel and the offset or ratio is applied to the other channel. For example, suppose CouplingEnabled is ON and CouplingMode is OFFSet. Furthermore, suppose channel 1 is operating at 2 kSa/s, and channel 2 is at 10 kSa/s. Setting Sample Rate Offset to 2.5 causes Channel 1 to remain at 2 Sa/s, and Channel 2 to be set to 4.5 Sa/s. As one channel's sample rate changes, the other channel's sample rate changes to maintain the specified coupling.
•If the sample rate coupling would cause either channel to exceed sample rate specifications for the present functions, the command will result in an error, and the sample rate will be set to its maximum or minimum limit for the channel.
property ratio

Sets offset ratio between channel sample rates when a two-channel instrument is in sample rate coupled mode RATio.

Access

ReadWrite

Type

float

Remarks
•Parameter : 0.001 to 1000, default 1.
•Typical Return : +7.500000000000000E-1
•When specifying OFFSet or RATio, the SOURce channel (SOURce1 or SOURce2) is used as the reference channel and the offset or ratio is applied to the other channel. For example, suppose the instrument is coupled in RATio mode. Furthermore, suppose channel 1 is operating at 2 kSa/s, and channel 2 is at 10 kSa/s. Setting Offset Ratio to 2.5 causes Channel 1 to remain at 2 kSa/s, and Channel 2 to be set to 5 kSa/s. As one channel's sample rate changes, the other channel's sample rate changes to maintain the specified coupling.
•If the sample rate coupling would cause either channel to exceed sample rate specifications for the present functions, the command will result in an error, and the sample rate will be set to its maximum or minimum limit for the channel.

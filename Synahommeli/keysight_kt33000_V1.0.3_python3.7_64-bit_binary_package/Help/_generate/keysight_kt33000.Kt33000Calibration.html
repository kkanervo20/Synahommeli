Kt33000Calibration

Access interface members using: driver.calibration
Where driver is a reference to the driver’s root interface returned by the constructor.
class Kt33000Calibration

Kt33000Calibration interface.

calibrate(self: keysight_kt33000.keysight_kt33000.Kt33000Calibration) → int

This method performs a calibration of the multimeter using the specified calibration value.

Returns

The method returns +0 (calibration passed) or +1 (calibration failed).

Return type

int

Remarks
•The instrument must be unlocked (i.e CalibrationSecureState must be OFF) to calibrate.
•Before running this command we need to set CalibrationState and CalibrationSignalValue.
•This query do the following:
•Increments the instrument’s calibration count.
•Modifies the volatile version of the calibration constants.
property count

Read Only. Returns the number of times instrument has been calibrated.

Access

ReadOnly

Type

int

Remarks
•Because the value increments for each calibration point, a complete calibration adds many counts.
•You can display count regardless of whether instrument is secured.
•This setting is non-volatile; it will not be changed by power cycling or Reset command.
property message

Stores a message of up to 40 characters in calibration memory.

Access

ReadWrite

Type

str

Remarks
•Parameter should be a quoted string up to 40 characters that may contain letters, numbers, spaces, and other common characters.
•Common messages include last calibration date, calibration due date, or contact information for calibration department.
•The instrument must be unlocked to store this string(i.e CalibrationSecureState must be OFF).
•The message can be read from the front panel or remote interface, regardless of whether the instrument is secured.
•Storing a calibration message overwrites the previous message.
•This setting is non-volatile; it will not be changed by power cycling or reset command.
property secure_code

The new security code to prevent accidental or unauthorized calibrations.

Access

WriteOnly

Type

str

Remarks
•Input is an unquoted string up to 12 characters that must start with letter (A-Z) and it may contain letters, numbers (0-9) and underscores.
•To change code: unsecure calibration memory with the old code, and then set new code.
•This setting is non-volatile; it will not be changed by power cycling or reset command.
property secure_state_enabled

Read Only. States whether the calibration security setting is secure or unsecure.

Access

ReadOnly

Type

bool

Remarks
Parameter : 0 (OFF) or 1 (ON). Default 1 (ON).
set_secure_state(self: keysight_kt33000.keysight_kt33000.Kt33000Calibration, mode: bool, code: str) → None

This method unsecures or secures the instrument for calibration. To calibrate, you must unsecure the instrument with the code.

Parameters

  • mode (bool) – True for secure, False for unsecure.

  • code (str) – An unquoted string up to 12 characters.

Returns

None

Remarks
•Front panel and remote interface calibration use same code. If you secure the instrument from one interface, use the same code to unsecure it from the other interface.
•This setting is non-volatile; it will not be changed by power cycling or Reset command.
property signal_value

Specifies the value of the known calibration signal.

Access

ReadWrite

Type

float

Remarks
•Parameter : Numeric, default 0.0
•Before running this command we need to set CalibrationState.
•This command sends the measured value from external instrument for calibration, such as DMM / Frequency Counter, etc. Not all types of calibration require external values, such as CalibrationState set to calibration step 1 which is essentially self-calibrated.
•This setting is non-volatile; it will not be changed by power cycling or reset command.
property state

The function generator’s internal state for each of the calibration steps to be performed.

Access

ReadWrite

Type

int

Remarks
•Parameter can be a whole number, default 1.
•The instrument must be unlocked (i.e CalibrationSecureState must be OFF) to calibrate.
•This setting is non-volatile; it will not be changed by power cycling or reset command.

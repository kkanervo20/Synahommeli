class Kt33000OutputChannelOutputCouple

Kt33000OutputChannelOutputCouple interface.

property enabled

Enables/disables frequency coupling between channels in a two-channel instrument.

Access

ReadWrite

Type

bool

Remarks
•Parameter : 0 (OFF) or 1 (ON). Default 0 (OFF).
•Specifying ON starts frequency coupling as specified by OutputCoupleMode.
•If the current offset or ratio, combined with the current frequency settings, would cause either frequency to exceed instrument specifications, the instrument will generate an error and the exceeded frequency will clip at its maximum or minimum value.
•If setting mode to RATIO and setting RATIO to 1.0 still exceeds the specifications of either channel (for example, channel 1 is a 3 MHz sine and channel 2 is a ramp, which cannot go that high), an error message will be generated and Coupling will be turned OFF.

property mode

Sets the type of frequency coupling between frequency coupled channels.

Access

ReadWrite

Type

CoupleMode

Remarks
•Parameter : Offset or Ratio. Default Ratio with ratio 1.0
•Typical Return : Offset or Ratio.
•Offset specifies a constant frequency offset between channels.
•Ratio specifies a constant ratio between the channels' frequencies.
•Power-on default for frequency coupling is OFF.
•Specifying SOURce1 or SOURce2 is irrelevant; either syntax sets the same coupling mode for both channels.
property offset

Sets the offset frequency when an instrument is in frequency coupled mode OFFSet.

Access

ReadWrite

Type

float

Remarks
•Parameter can be a number between plus and minus the instrument's maximum frequency for the waveform; default 0.
•Typical Return : +7.650000000000000E+05
•The SOURce channel (SOURce1 or SOURce2) is used as the reference channel and the Offset is applied to the other channel. For example, suppose the instrument is set to CoupleEnabled ON and CoupleMode is set to Offset (frequency offset mode active), and channel 1 is currently operating at 1.0 MHz .Setting Offset to 500 KHz will cause channel 1 to remain at 1.0 MHz, and channel 2 to be set to 1.5 MHz. As the frequency of either channel is changed, the frequency of the other channel will change to maintain the 500 kHz offset.
 •If the frequency coupling would cause either channel to exceed instrument frequency specifications for the present functions, the command will result in an error, and the frequency will be set to its maximum or minimum limit for that channel.
 •Frequency coupling is not valid with arbitrary waveforms, but you can couple the sample rate of arbitrary waveforms.
property ratio

Sets offset ratio between channel frequencies in frequency coupled mode RATio.

Access

ReadWrite

Type

float

Remarks
•Parameter : 0.001 to 1000, default 1.
•The SOURce channel (SOURce1 or SOURce2) is used as the reference channel and the RATIO is applied to the other channel. For example, suppose the instrument is set to CoupleEnabled ON and CoupleMode is set to Ratio. Furthermore, suppose channel 1 is currently operating at 2 kHz, and channel 2 is at 10 kHz. Setting Ratio to 2.5 will cause channel 1 to remain at 2 kHz, and Channel 2 to be set to 5 kHz. As the frequency of either channel is changed, the frequency of the other channel will change to maintain the 2.5 ratio.
•If the frequency coupling would cause either channel to exceed instrument frequency specifications for the present functions, the command will result in an error, and the frequency will be set to its maximum or minimum limit for that channel.
•Frequency coupling is not valid with arbitrary waveforms, but you can couple the sample rate of arbitrary waveforms.
